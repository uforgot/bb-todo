# 업데이트 빵빵 — 의존성 관측 대시보드

내가 의존하고 있는 라이브러리들이 언제 어떻게 좋아지는지를 한 화면에서 본다. 보안 패치 중심의 디펜다봇·리노베이트와는 결이 달라 — "내가 기다리는 모델/라이브러리"의 발전 신호를 잡는 큐레이션 도구.

## 자리

할일빵빵(bb-todo) 안에 메뉴로 둔다. 비비 어드민은 개인 데이터 보관소라 결이 안 맞음. 작업/생산성 결의 도구는 bb-todo가 단일 입구가 되게.

## 흐름

1. **스캔** — 등록된 레포 목록의 `package.json`, `requirements.txt`, `uv.lock`, `pyproject.toml` 같은 매니페스트를 정기적으로 긁어 의존성 통합 목록 구성.
2. **버전 조회** — NPM과 PyPI 레지스트리에 현재 vs 최신 버전 조회. 차이를 메이저 / 마이너 / 패치로 구분.
3. **큐레이션** — 핵심 라이브러리(insightface, ElevenLabs SDK, openclaw, ably 등)에 별표. 별표 없으면 조용히 숨김 처리해서 잔소리 안 되게.
4. **릴리스 노트** — GitHub 릴리스나 changelog 가져와 카드에 미리보기. AI로 "이 업데이트가 내 프로젝트에 어떻게 영향 주는지" 한 줄 생성.
5. **알림** — 별표 라이브러리에 메이저 업데이트 뜨면 Discord나 봇으로 한 마디. 그 외는 대시보드에서만 조용히 노출.

## 차별점 (왜 만드냐)

- 기존 도구는 회사용 보안 패치 관점. 개인 메이커가 "기다리던 모델이 진화했다"를 알고 기능을 설계하는 도구는 없음.
- 봇 시리즈(어디야/건강해 빵빵)와 같은 결로, 데이터를 잔소리가 아닌 신호로 만드는 게 핵심.

## 단계 (대략)

1. bb-todo 서버에 manifest scanner 추가 (지정된 레포 경로 목록을 주기적으로 훑음).
2. NPM/PyPI 버전 조회 + 캐싱.
3. bb-app 또는 bb-todo 웹에 의존성 페이지 UI.
4. 별표 기능 + 큐레이션 저장.
5. GitHub 릴리스 노트 가져오기.
6. AI 요약 (Claude/GPT로 한 줄 영향 평가).
7. 알림 정책 (별표 + 메이저만 → 봇 메시지).

## 열린 질문

- 매니페스트 위치를 어떻게 등록할까. 폴더 경로 리스트만으로 충분한지, 별도 설정이 필요한지.
- 봇이 보낼 알림 빈도 — 하루 한 번 다이제스트가 자연스러울 듯.
- 비공개 레포의 깃허브 토큰 관리.
