# 어디야 빵빵 — 위치 정보 통합 플랜

보이스 앱으로 말할 때만 현재 위치를 같이 실어 보내서, 봇이 "어디서 묻는지" 알고 답하게 한다.

## 목표

- 위치 트리거를 보이스 앱 발화 시점에만 한정. heartbeat나 백그라운드 추적은 하지 않음.
- 좌표만 보내지 않고, 사용자가 미리 등록한 장소 이름으로 변환해 의미 있는 컨텍스트로 만든다 (e.g. `집`, `회사`, `장모님댁`).
- 등록되지 않은 좌표는 Google Places로 보강. 그것도 안 되면 좌표 그대로 보낸다.

## 범위 (제외 항목)

- 항상 추적 / 백그라운드 위치 사용 안 함.
- 가족 폰 트래킹/지오펜스는 별개 프로젝트. 여기는 본인 폰만.
- 카메라/이미지 동반 전송은 별도 항목.

## 컴포넌트

### 1. iOS — 위치 권한 + 좌표 캡처

- `Info.plist`: `NSLocationWhenInUseUsageDescription` 추가. "보이스 대화 시 현재 위치를 빵빵에게 알리는 데 사용." 정도.
- `Services/LocationService.swift` (신규): `CLLocationManager` 래퍼. 권한 요청, 1회성 `requestLocation()`.
- `VoiceView` `onAppear`에서 권한 요청. 거부 시 위치 없이 진행.
- `VoiceService.publishRequest`에서 마지막으로 받은 좌표를 페이로드 `location: { lat, lng, accuracy, ts }`에 끼워 송신.

### 2. 데이터 저장소 — bb-admin이 단일 소유

- `voice-config.json`에는 **places 데이터를 넣지 않는다**. 데이터가 커질 수 있고 운영용 설정과 섞이는 게 싫음.
- bb-admin에 `places` 전용 스토어를 둔다 (얼굴 인식 데이터셋이 이미 bb-admin에서 관리되는 것과 같은 패턴).
  - 파일 위치 후보: `bb-admin/data/places.json` 또는 기존 face-admin과 같은 storage layer.
  - 스키마: `{ name, lat, lng, radiusM, createdAt }[]`.

### 3. 서버 — 좌표를 이름으로 변환

- `voice-bridge.js`가 bb-admin API에서 places 리스트를 받아옴 (캐싱).
- 들어온 좌표를 places 리스트와 비교. 반경 안이면 이름으로 치환.
- 매칭 실패 시 Google Places API (openclaw 설치 시 등록된 `GOOGLE_PLACES_API_KEY` 재사용)로 `goplaces nearby` 호출 → 가장 가까운 POI 이름 사용.
- 그것도 실패하면 `"위도 X 경도 Y"` 그대로 컨텍스트에 첨부.

### 4. 등록 UI — bb-admin 페이지 + API

- `bb-admin/src/app/places/` 페이지 신규. face-admin 페이지와 같은 결의 메뉴로 추가.
  - 목록 보기 / 추가 / 수정 / 삭제.
  - 추가 시 좌표는 직접 입력하거나 지도에서 찍기 (지도는 phase 2).
- `bb-admin/src/app/api/places/route.ts`:
  - `GET` — 전체 리스트.
  - `POST` — 신규 등록 `{ name, lat, lng, radiusM? }`.
  - `PATCH /[id]` — 수정.
  - `DELETE /[id]` — 삭제.
- 이 API를 voice-bridge가 사용. bb-app은 "현재 좌표를 빠르게 등록" 버튼 정도만 두고 본 관리는 bb-admin에서.

### 5. 어시스턴트에 전달

- 봇 컨텍스트에 `현재 위치: {장소이름}` 한 줄만 prepend. 좌표를 모델에 직접 던지지 않음 (필요 시 디버그 모드에서만).
- 등록 안 된 좌표: `현재 위치: 미등록 (서울 강남구 역삼동 근처)` 식으로 Google Places 결과 한 줄 추가.

## 단계별 작업

1. bb-admin에 places 스토어 + GET/POST API 추가 (face-admin 패턴 미러).
2. bb-admin places 페이지 UI (목록/추가/삭제).
3. iOS `LocationService` 추가, Info.plist 키 등록.
4. `VoiceView` 권한 흐름 + 좌표 캡처.
5. `publishRequest` 페이로드 확장.
6. `voice-bridge` places 매칭 로직 (bb-admin API 호출 + 캐싱).
7. Google Places nearby fallback.
8. 어시스턴트 system message에 위치 라인 끼우기.
9. (선택) bb-app에 "지금 여기 등록" 단축 버튼.

## 열린 질문

- 정확도: GPS (수 미터) vs Wi-Fi (수십~수백 미터) 중 어느 단계까지. 등록 장소 반경 디폴트 100m면 Wi-Fi 정확도로 충분할 가능성 큼.
- `places` 편집은 앱 UI에서만 할지, voice-config.json 직접 편집도 허용할지.
- 좌표 로그를 서버에 남길지 — 디버그 외 보관하지 않는 게 깔끔.
